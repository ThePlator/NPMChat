## 🛠 Project Standards

- **TypeScript:** Use proper types and interfaces. Avoid `any` whenever possible (See [Issue #71](https://github.com/ThePlator/NPMChat/issues/71)).
- **Styling:** Follow the Neo-Brutalist design system.
- **Commits:** Use clear, descriptive commit messages.
- **PRs:** Link your Pull Requests to the corresponding Issue.
